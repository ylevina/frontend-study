1. Обьяснить своими словами разницу между обьявлением переменных через var, let и const.

    var переменные можно переназначать, "не поддерживает" константных значений. Зона видимости ограничивается функцией в которой была обьвленна эта переменная. Например, если внутри функции есть цикл внутри которого обьявляются переменные, то эти переменные будут видимы/доступны после блока цикла но внутри этой же функции.

    let переменные можно так же переназначать, но зона видимости ограничивается блоком, тоесть { }. На примере с циклом, если попробовать обратиться к переменной из цикла вне цикла то будет референс еррор по причине undefined значения. 

    const используется для значений которые константны, не будут изменяться, например год рождения. С зонами видимости все так же как и у let.
    

2. Почему объявлять переменную через var считается плохим тоном?

    var старый способ обьявления переменных. Также существует нюанс связанный с зоной видимости и возможностью повторного переназначения var переменных. По этим причинам придумали и добавили let и const для разделения изменяемых переменных и констант, а так же что бы пофиксить проблему c зонами видимости. Начиная с ES2015 (ES6) появились let и const, var поддерживается для обратной совместимости.
